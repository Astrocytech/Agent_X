# Customer-Facing Explanation Contract

## Purpose
Define the format and content requirements for customer-facing explanations generated by the BenchCore system. These explanations communicate why a particular suggestion, command, or transformation was made.

## Source
BENCHCORE-DOC-023

## Explanation Format
Each customer-facing explanation MUST contain the following sections in order:

```
## Summary
One or two sentences summarizing the action or suggestion in plain language.

## What Changed
A bulleted list of specific changes, each including:
- Field or element name
- Previous value (if applicable)
- New value

## Why
A plain-language explanation of the rationale, avoiding technical jargon.

## Impact
A description of what effect this change will have on the output or workflow.

## Source Reference
A reference to the specific rule, policy, or schema that drove the change.
```

## Required Sections
1. **Summary**: Always required. Max 280 characters.
2. **What Changed**: Always required. At least one bullet.
3. **Why**: Always required. Max 500 characters.
4. **Impact**: Always required. Max 500 characters.
5. **Source Reference**: Always required. Must include a document ID (e.g., BENCHCORE-DOC-xxx).

## Language Guidelines
- Use plain, non-technical English. Avoid acronyms without explanation.
- Use active voice where possible.
- Do not use phrases like "the AI determined" or "the system decided." Instead, use rule-based language: "based on the formatting policy" or "to match the required field format."
- Keep each section concise. Do not repeat information across sections.
- Assume the reader is a content producer or editor, not a software engineer.

## Example
```
## Summary
A keyword delimiter was changed to match the required format.

## What Changed
- Field: delimiter
- Previous value: /
- New value: :

## Why
The system uses ":" as the standard delimiter. The previous delimiter "/" is not supported.

## Impact
This change ensures the cue will be processed correctly by the production system.

## Source Reference
BENCHCORE-DOC-020
```

## Constraints
- No marketing language or promotional content.
- No references to internal system architecture or implementation details.
- No technical debugging information (stack traces, error codes, etc.).
