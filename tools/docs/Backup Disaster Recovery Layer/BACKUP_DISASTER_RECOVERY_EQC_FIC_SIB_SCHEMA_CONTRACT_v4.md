## 1. EQC (External Quality Contract)

### 1.1 BackupRecord Contract

```python
@dataclass
class BackupRecord:
    schema_version: str   # = "1.0"
    backup_id: str        # unique, generated by new_id("bk")
    category: str         # one of BC_* constants
    status: str           # BK_PENDING | BK_COMPLETED | BK_FAILED
    source_paths: list[str]
    backup_paths: list[str]
    checksum: str         # SHA-256 hex digest (canonical JSON)
    size_bytes: int       # total size of backed-up files
    created_at: str       # ISO 8601 UTC
    completed_at: str     # ISO 8601 UTC (set on completion/failure)
    warnings: list[str]
    errors: list[str]
```

### 1.2 BackupManifest Contract

```python
@dataclass
class BackupManifest:
    schema_version: str   # = "1.0"
    manifest_id: str      # unique, generated by new_id("bkm")
    backups: list[BackupRecord]
    total_backups: int    # derived = len(backups)
    total_size_bytes: int # derived = sum(b.size_bytes)
    created_at: str       # ISO 8601 UTC
    warnings: list[str]
    errors: list[str]
```

## 3. SIB (State Integrity Boundary)

### 3.1 Persistent State Files

| File | Format | Location |
|---|---|---|
| backups.jsonl | JSONL (append-only) | `.agentx-init/backup/` |
| manifest.json | JSON (atomic overwrite) | `.agentx-init/backup/` |
| .backup.lock | Empty file (flock) | `.agentx-init/backup/` |

### 3.2 Integrity Rules

1. Each JSONL line is a canonical JSON object with no trailing whitespace
2. Manifest total_backups must equal len(backups)
3. Manifest total_size_bytes must equal sum of all record size_bytes
4. Checksum is SHA-256 of record's canonical JSON
5. Atomic writes use temp file + os.replace
