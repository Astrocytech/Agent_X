# Naming Conventions

Production code, tests, and documentation follow the `snake_case` convention for Python, YAML, and JSON files.

- Source files: `snake_case.py`
- Test files: `test_<component>__<behavior>.py`
- Schema files: `<name>.schema.json`
- Documentation: `UPPER_SNAKE_CASE.md`
- YAML configuration: `lower-kebab-case.yaml`
